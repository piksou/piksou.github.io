Linux Server 安装后启动卡在 Loading initial ramdisk 或 EFI stub: measured initrd data into PCR 9 的解决办法

发表于 2024-10-24 分类于 Notes 阅读次数：
本文字数： 344 阅读时长 ≈ 1 分钟

记录下遇到的坑。

问题描述

新装 Ubuntu Server 或者 Debian 后，无法启动进入系统，其中Ubuntu卡在如下错误信息处：

Debian 卡在如下错误信息处：

Ubuntu 短暂 Workaround

启动时调出 GRUB 的选择界面，选择 Advanced Options，然后进入 Recovery 模式。进入后选择 resume 选项以正常启动系统。注意，Debian无法按此方式启动。

解决办法

在 GRUB 启动选择菜单中按 ‘e’ 键， 在 linux 内核启动选项后添加参数dis_ucode_ldr。

启动后，修改/etc/default/grub文件,

将参数GRUB_CMDLINE_LINUX="" 修改为GRUB_CMDLINE_LINUX="dis_ucode_ldr"

修改完成后，用 sudo update-grub 更新 grub。然后重启。

原因

Intel CPU 的微指令（Microcode）出了bug。Debian的帮助文档这么说

Very rarely, it is possible for a kernel or a microcode update bug to cause boot issues (hangs or resets at the very beginning of the boot process) on specific processor models.

后续处理

关键词：Updating CPU microcode within Ubuntu/Debian

或者在系统内部运行 fwupdmgr get-upgrades 查看是否有固件更新。如有，可用 fwupdmgr update 更新。

吐槽

“Very rarely” 我也能遇到，估计如果是搞实验物理，应该就是走到哪里仪器坏哪里的那种？
