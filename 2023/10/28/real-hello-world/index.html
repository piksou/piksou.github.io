A Real Hello World

发表于 2023-10-28 阅读次数：
本文字数： 1.4k 阅读时长 ≈ 5 分钟

那个是假的 Hello World, 这个才是真的。

1. 什么是 Hexo

据观察，Hexo 是一个全是老中在用的静态博客框架。比较多人用的主题里，甚至有直接填网站备案号的选项，看着都非常亲切。

2. 为什么要用 Hexo

其实我本来想用 Vitepress。 根据 这篇文章 Vitepress 好看又简洁，但一来这个什么鬼Vue3我不会，二来看到配置一个博客的界面要搞这么多东西，我也不想学。

然后我又对比了一下 Hexo 和 Hugo，明显 Hugo 的使用人群更加国际化，但是一看文档，上来就让你安装 Go、Dart Sass, 好像自定义主题还要用到 Node.js，我就不想用了。哪有 Node.js 一把梭好。我都把博客放在 GitHub Page 上了，你这一点点编译加载时间的优化在国内的网络下，不是开玩笑吗。（中途还去注册了个Vercel号，最后闲置了）

3. 中间的转折

挑主题，好吧，我本来想用 mashiro，看了半晚上发现文档不是很全，算了。然后是NexT，看文档看到巨晚，一条一条看这个配置文件，终于大概懂了hexo主题中的套路。一不小心，中间看到了 Fluid，然后就去捣鼓，甚至连配置文件，背景图片都下载好了，还配好了字体和favicon， 最后发现算了，还是不要搞那么多花里胡哨的东西。不如就直接Next吧，烂大街的东西用着也不会错。

4. 为什么要用 GitHub Page

因为我不想花钱维护域名，或者说没找到合适的长期持有的域名（手头只有年抛产品）。

5. 还是得集成 Github Action

主要是要出差，不想要出差的公用的电脑上部署 git 之类。

6. 换了个字体，加了个AddToAny

字体用了 霞鹜文楷， 网页版在 这里。主要做法是在themes/next/layout/_layout.swig文件，在 head 中插入字体的CSS文件链接。具体可以参考这篇文章和这篇文章。如果用的是Fluid，还可以参考这篇文章。不过，字体设置直接在配置文件里面搞就好了，最后改了_layout.swig文件，不如这个优雅。AddToAny 本来想着拿来分享的，好像没什么用。不过，还是留着吧，大概就是网页最后有个文字为Share的链接。增加的内容这里保存下。

7. 改主题就有了巨坑

1. 开始

启动文件夹，写配置，加载插件。

2. 把 about 和 categories 写了， 搞个local search

加上type字段就行。见文档。加的配置里面的 local search 搜索，不知道有没有用。

3. Git Action 一把梭

这两种。我要改字体，并且还要结合 Github Action，那自然第一种就不能用了，否则我还得在Github Action环境里面的node_modules里面改。但是你第二种直接 git clone 然后就在一个 git项目里面套了另一个，theme/next 里面的东西都加不进去。查了下Github Action的输出，是layout报错，提示各种找不到layout。最后的解决办法，还是直接下载仓库文件直接复制进去。然后就好了。可能可以用 git submodule 来解决我遇到的这个问题，但是我懒得继续研究了。

上面的那个自动部署脚本，有一点老，发现node好像还是用的12，就搜了一下别人的，加上了部署node版本的步骤，但是最后部署不了。犯了两个错误，一个是typo，略去不表；另一个是，hexo deploy 说找不到 type: git 。好吧， 是忘了加 hexo-deployer-git 了。

4. 其它

Github Page里面 Settings > Pages > Build and deployment > Source， 选 Deploy from a branch，被官方的误导了下。
