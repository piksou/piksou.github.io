终端设置指南

发表于 2025-01-14 分类于 Notes 阅读次数：
本文字数： 539 阅读时长 ≈ 2 分钟

简单记录下设置的一些小技巧。

最近用终端比较多，主要是Windows下的 Windows Terminal 和 Git for windows 带的简化版本 bash。当然WSL也挺好用的，就是很多时候用不到那么重的功能。这里记录下自己设置的一些小技巧，方便以后使用。

Windows Terminal 添加 Miniconda Powershell 环境

  1. 打开 Windows Terminal 设置页面，拖到最下面，选择 添加新配置文件；
  2. 选择 新建空配置文件；
  3. 选择命令行，输入 %windir%\System32\WindowsPowerShell\v1.0\powershell.exe -ExecutionPolicy ByPass -NoExit -Command "& 'C:\Users\Username\Miniconda3\shell\condabin\conda-hook.ps1' ; conda activate 'C:\Users\Username\Miniconda3' "，注意把Username替换成自己的用户名；
  4. 启动目录设置为 %USERPROFILE%；
  5. 更改你需要的名称和图标即可。

Git for Windows 添加 conda

  1. 打开bash, 安装的miniconda文件夹，然后去到类似 /c/Users/Username/Miniconda3/etc/profile.d/的文件夹内。
  2. 在bash里运行 echo ". '${PWD}'/conda.sh" >> ~/.bashrc。
  3. 然后你就能在bash里用conda了。

PS: 其实不如miniconda带的另外两个终端好用。

Git for Windows 添加 rsync

参考此Gist。以下为摘录：

  1. 打开 https://repo.msys2.org/msys/x86_64/；
  2. 搜索rsync，下载.pkg.tar.zst的压缩包。
  3. 搜索libxxhash，下载.pkg.tar.zst的压缩包。
  4. （我这里不需要）搜索libopenssl，下载.pkg.tar.zst的压缩包。
  5. 去https://github.com/facebook/zstd/releases下载win64的文件，解压出zstd.exe。
  6. 用 tar --use-compress-program="zstd -d" -xvf 文件名.pkg.tar.zst 解压文件。
  7. 将解压出来的文件夹内的 /usr/bin 里面的rsync.exe 以及 msys-xxhash-0.dll 复制到 C:\Program Files\Git\usr\bin。

PS: Windows下有个叫做 robocopy 的命令，也有点类似。
